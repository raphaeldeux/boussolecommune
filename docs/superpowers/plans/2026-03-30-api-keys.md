# Gestion des clés API par ville — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Permettre à chaque gestionnaire de ville de configurer ses propres clés API (INSEE + Mistral) depuis l'espace admin, sans modifier le `.env`.

**Architecture:** 3 nouvelles colonnes sur la table `villes` (`insee_api_key`, `mistral_api_key`, `mistral_model`). Les fetchers et `ai_service.py` acceptent la clé en paramètre avec fallback `os.environ`. Une page dédiée `/admin/integrations` est accessible aux gestionnaires ; la fiche commune (super_admin) reçoit une section API.

**Tech Stack:** Flask 3.0, psycopg2, Jinja2 — aucune nouvelle dépendance.

```python
def _get_bearer_token(api_key: str):
    """Obtient un Bearer token OAuth2 depuis l'API INSEE."""
    if not api_key or ":" not in api_key:
        raise ValueError(
            "INSEE_API_KEY invalide. Format attendu : 'consumer_key:consumer_secret'. "
            "Inscription gratuite sur https://api.insee.fr/catalogue/"
        )
    credentials = base64.b64encode(api_key.encode()).decode()
    resp = requests.post(
        TOKEN_URL,
        headers={
            "Authorization": f"Basic {credentials}",
            "Content-Type": "application/x-www-form-urlencoded",
        },
        data="grant_type=client_credentials",
        timeout=TIMEOUT,
    )
    resp.raise_for_status()
    return resp.json()["access_token"]
```

```python
def _appel_mistral(prompt: str, api_key: str = None, model: str = None) -> str:
    """Appel Mistral AI. Contexte 128k tokens."""
    _key = (api_key or "").strip() or MISTRAL_API_KEY
    _model = (model or "").strip() or MISTRAL_MODEL
    response = requests.post(
        MISTRAL_API_URL,
        headers={
            "Authorization": f"Bearer {_key}",
            "Content-Type": "application/json",
        },
        json={
            "model": _model,
            "messages": [{"role": "user", "content": prompt}],
            "temperature": 0.3,
        },
        timeout=180,
    )
    response.raise_for_status()
    return response.json()["choices"][0]["message"]["content"].strip()
```

```python
# ── Intégrations API (gestionnaire + super_admin) ─────────────────────────────

@bp.route("/integrations", methods=["GET", "POST"])
@login_required
def integrations():
    """Page de configuration des clés API pour la ville en session."""
    import os as _os
    ville = _get_current_ville()
    if not ville:
        flash("Aucune commune sélectionnée.", "danger")
        return redirect(url_for("admin.upload"))

    if request.method == "POST":
        insee_api_key = request.form.get("insee_api_key", "").strip() or None
        mistral_api_key = request.form.get("mistral_api_key", "").strip() or None
        mistral_model = request.form.get("mistral_model", "").strip() or None

        # Avertissement non-bloquant si format INSEE incorrect
        if insee_api_key and ":" not in insee_api_key:
            flash("Format de clé INSEE invalide (attendu : consumer_key:consumer_secret).", "warning")

        ville_model.update_api_keys(
            ville["id"],
            insee_api_key=insee_api_key,
            mistral_api_key=mistral_api_key,
            mistral_model=mistral_model,
        )
        flash("Clés API enregistrées.", "success")
        return redirect(url_for("admin.integrations"))

    has_env_insee = bool(_os.environ.get("INSEE_API_KEY"))
    has_env_mistral = bool(_os.environ.get("MISTRAL_API_KEY"))
    # Recharger la ville pour avoir les valeurs à jour
    ville = ville_model.get_by_id(ville["id"])
    return render_template(
        "admin/integrations.html",
        ville=ville,
        has_env_insee=has_env_insee,
        has_env_mistral=has_env_mistral,
    )
```

## Notes importantes pour l'implémenteur

### Thread safety (Task 5, Steps 3 et 4)

Les routes `conseil_generer_resume` et `conseil_analyser_note` lancent un thread avec `threading.Thread`. Le contexte Flask (request, session) **n'est pas disponible dans le thread**. La clé API doit être capturée dans une variable locale **avant** la définition de `_run()` :

```python
# ✅ Correct — capture avant le thread
mistral_key = (ville.get("mistral_api_key") or "").strip()
def _run():
    generer_resume(pdf_path, api_key=mistral_key, ...)
```

```python
# ❌ Incorrect — accès session dans le thread
def _run():
    ville = ville_model.get_by_id(session.get("admin_ville_id"))  # crash hors contexte
```

### Valeurs None vs chaîne vide

Dans `update_api_keys()`, une chaîne vide (`""`) est convertie en `None`. Cela permet à l'utilisateur d'**effacer** une clé en vidant le champ, ce qui remet le fallback env en service.

### Backward compatibility

Les fetchers et `ai_service.py` gardent `api_key=None` comme valeur par défaut. Si personne ne passe de clé (ancien code), ils tombent en fallback env. Aucune régression.
